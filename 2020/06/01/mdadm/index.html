mdraid check

grub boot failed because replace mdadm root device

  • CentOS 7 ISO boot
    • trouble shotting –> resuce mode –> 1.continue/2.read-only mode/3.skip to shell/4.quit(reboot)
      • select 1 continue, but the rescue mode show “not found linux partition”
        • detect the mdadm raid –> mdadm –examine –scan >> /etc/mdadm.conf
          • show the “/dev/md/efi_boot and /dev/md/root”
        • mdadm –assemble –scan /dev/md/efi_boot
        • mdadm –assemble –scan /dev/md/root
        • cat /proc/mdadm show md126(efi) and md127(root) and the new device not in the raid
        • show “md127 : active (auto-read-only)”
          • mdadm –readwrite /dev/md1
        • mount /dev/md127 /mnt/sysimage
        • mount /dev/md126 /mnt/sysimage/boot/efi
        • mount -o bind /proc /mnt/sysimage/proc
        • mount -o bind /sys /mnt/sysimage/sys
        • mount -o bind /dev /mnt/sysimage/dev
        • chroot /mnt/sysimage
          • legacy boot mode
            • grub2-mkconfig -o /boot/efi/EFI/centos/grub.cfg
            • grub2-install /dev/sda(which one you replaced)
        • or you could copy kernel rpm package(iso) and reinstall

About linux soft raid5

The stripe_cache_size only be set to 16~2048, you can’ t disable it, the write hole will cause data loss
So the raid10 or raid0 is the better choice than linux soft raid 4/5/6
If you add the journal device, the too many performance impact in the raid device

How to backup mdraid info by dd

In mdraid superblock, you could see bitmap starts at 8 x 512B=4096
You could see “bitm” magic num start at 0x1000=4096

You could see the data update from 0x1200, so you could backup first 4608 Bytes in raid1
dd if=/dev/sdb1 of=sdb1_superblock bs=1 count=4608

mdadm raid5 xor

don’t need to read each block, in raid5 4k data, only 2 device could be read, more device means more iops
For reduce read amplification, it will compare the old data size (i[old0] > i[2]+..i[n] or i[old0] < i[2]+..i[n])
rmw:read-modified-write, i[old0] < i[2]+..i[n], only read i[old0], just read i[old0] and the old parity c[old]
rcw:read-construct-write, i[old0] > i[2]+..i[n], only read i[2]..i[n], just read i[2]..i[n] and the old parity c[old]
